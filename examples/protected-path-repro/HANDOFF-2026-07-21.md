# Handoff — Protected-Path Repro (Kevin Neilson forum challenge)

Written 2026-07-21 end of session, to be picked up 2026-07-22. This is a
**hypothesis, not a confirmed finding** — do not send anything to Kevin

## Hipótesis actual

**Cursor CLI (Windows) does not block tool execution when a `preToolUse`/
`beforeShellExecution` hook returns `deny` (exit code 2), while Cursor
Desktop IDE (Windows) does.** If confirmed clean, this is exactly the
reproducible "denial bypassed" case Kevin invited in the forum thread.

## Lo que ya pasó (cronología)

1. Construimos `protected-path-guard.js`: fail-closed hook que bloquea
   Write/Delete sobre `docs/protected/EVAL-RUBRIC.md` vía `preToolUse`,
   `beforeShellExecution`, `beforeMCPExecution`, más `permissions.deny` en
   `.cursor/cli.json`. 17/17 casos pasan localmente (`node test-local-guard.js`).
2. Primera corrida de CLI: **inválida** — el agente estaba parado en una
   carpeta/repo distinto ("CursorCLI"), sin nuestro `hooks.json`. Nada de
   lo que pasó ahí cuenta.
3. Primera corrida de IDE: bloqueó correctamente escritura/borrado/shell,
   y permitió lectura + escritura de archivo no relacionado — confirmado
   con `hooks.log` real, no solo el reporte del agente. Este resultado
   **sí es válido**.
4. Segunda corrida (ambos, ruta correcta `C:\LabCursorTest\examples\
   protected-path-repro`, baseline SHA256 `10220330...` confirmado antes
   de empezar):
   - **IDE**: bloqueó los 8 prompts, incluidos los de control (#7 lectura,
     #8 `scratch.md`) — contradice el comportamiento correcto del punto 3.
   - **CLI**: casi todo "OK" (edit, delete, rm, redirect, sed, glob,
     lectura mostró "pwned", scratch.md creado).
5. Pedí el `hooks.log` real del CLI. Resultado: **10 entradas `[DENY]`
   consecutivas**, todas con el mismo error:
   `JSON parse error: Unexpected token '﻿'` — un BOM UTF-8 (`U+FEFF`) que
   Cursor antepone al payload de stdin en Windows, y que `JSON.parse` sin
   sanitizar rechaza.
6. Esto explica el punto 4 completo: el BOM rompe el parseo en *todas*
   las llamadas, así que el hook (fail-closed, correctamente) denegó todo
   — tanto en IDE como, aparentemente, en CLI. La diferencia es que **el
   IDE respetó ese deny universal (bloqueó todo, incluidos los controles
   — coherente)**, y **el CLI lo ignoró y ejecutó los comandos de todas
   formas** (edit, delete, rm, sed, redirect, glob, todos "OK" pese a que
   el hook dijo deny las 10 veces).
7. Corregí el bug del BOM en `protected-path-guard.js` (strip de
   `﻿` antes de `JSON.parse`) y agregué 2 casos de regresión al
   harness local. 17/17 pasan. **Este fix está commiteado en este repo
   local pero AÚN NO empujado al repo público** al momento de escribir
   esto — confirmar que el push se completó antes de seguir.
