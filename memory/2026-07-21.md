2. **Resolución de Configuración de Hooks (revisado tras verificación contra doc oficial)**:
   - Confirmado por `https://cursor.com/docs/agent/hooks`: 4 capas con prioridad **Enterprise → Team → Project → User** (`C:\ProgramData\Cursor\hooks.json` en Windows / `/etc/cursor/hooks.json` en Linux → Team → `.cursor/hooks.json` → `~/.cursor/hooks.json`).
   - `preToolUse` es un Agent hook documentado oficialmente.
   - **Corrección:** la hipótesis original de que "Cursor IDE lee solo `.cursor/hooks.json` de workspace, mientras que Cursor CLI lee solo hooks globales y mapea `beforeSubmitPrompt`/`afterShellExecution`" **no está confirmada por la documentación oficial** — la doc no distingue explícitamente el soporte de `preToolUse` entre IDE y CLI. Se retiró esa afirmación del post público y quedó como pregunta abierta a @CursorDev en el foro en vez de publicarse como hecho.
