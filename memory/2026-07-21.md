# Registro de Memoria - 21 de Julio de 2026

## 🎯 Objetivo Cumplido
Creación y validación del entorno mínimo de prueba para **Cursor Hooks (`preToolUse`)** en `C:\LabCursorTest`.

## 🛠️ Lecciones Técnicas y Arquitectura Aprendida
1. **Configuración de Hooks**:
   - Archivo `.cursor/hooks.json` declara las etapas de ciclo de vida (`preToolUse`, `postToolUse`, etc.).
   - `matcher`: Expresión regular para filtrar herramientas objetivo.
   - `timeout`: Tiempo límite en segundos.

2. **Manejo de Entrada/Salida por Stdin/Stdout**:
   - Cursor transmite la llamada a la herramienta vía `stdin` en formato JSON (`tool_name`, `tool_input`, `conversation_id`).
   - El script debe leer `stdin` hasta completar (EOF).

3. **Códigos de Salida e Intercepción**:
   - **`Exit Code 0`**: Permite la ejecución de la herramienta por parte del agente (`{"permission": "allow"}`).
   - **`Exit Code 2`**: Deniega la ejecución e informa el motivo en `stdout` (`{"permission": "deny", "reason": "..."}`).

4. **Verificación Automatizada**:
   - Suite `test-hook-runner.js` simula el proceso `spawn` enviando payloads por `stdin` y validando `exitCode` y los logs generados en `.cursor/hooks/hooks.log`.
